## Exercise 1: The Decrementing Jump Loop

Analyze the time and space complexity of the following function.

```python
def lookup_jump(n):
    count = 0
    i = n
    while i > 1:
        count += 1
        i = i // 2  # Integer division (halving i)
    return count

```

### Step-by-Step Solution:

1. **Identify the loop behavior:** The loop variable `i` starts at $n$. In every single iteration, `i` is halved (`i // 2`).
2. **Determine the termination condition:** The loop runs until `i` is no longer greater than 1.
3. **Formulate the math:** We are asking: *"How many times ($k$) can we divide $n$ by 2 before reaching 1?"*

$$\frac{n}{2^k} = 1 \implies n = 2^k \implies k = \log_2(n)$$


4. **Final Complexities:**
* **Time Complexity:** $O(\log n)$ — Logarithmic growth.
* **Space Complexity:** $O(1)$ — Only a few primitive integer trackers are held in memory.

## Exercise 2: The Constrained Matrix Scan

Analyze the time and space complexity of the following function.

```python
def process_grid(matrix):
    rows = len(matrix)
    cols = len(matrix[0]) if rows > 0 else 0
    
    # Loop A: Fixed constraint execution
    for i in range(min(rows, 100)):
        print(matrix[i][0])
        
    # Loop B: Complete dependent execution
    for r in range(rows):
        for c in range(cols):
            matrix[r][c] *= 2

```

### Step-by-Step Solution:

1. **Analyze Loop A:** This loop runs from 0 to either `rows` or 100, whichever is smaller. Because it is strictly capped at a maximum ceiling of 100 iterations, its runtime is fundamentally constant: $O(1)$.
2. **Analyze Loop B:** This contains a nested loop architecture. The outer loop runs $R$ times (rows), and the inner loop runs $C$ times (columns) for every single row. Total operations = $R \times C$.
3. **Combine and drop lower-order terms:** Total Time $T = O(1) + O(R \times C)$. The dominant term is clearly the matrix dimension multiplication.
4. **Final Complexities:**
* **Time Complexity:** $O(R \times C)$ (or $O(n)$ where $n$ is the total number of cells in the grid).
* **Space Complexity:** $O(1)$ Auxiliary Space — The matrix is modified directly "in-place" without generating new collections.

## Exercise 3: Two-Pointer Target Search

Analyze the **Best-Case ($\Omega$)** and **Worst-Case ($O$)** time complexity of this array scanning algorithm.

```python
def possesses_pair_sum(sorted_arr, target):
    left = 0
    right = len(sorted_arr) - 1
    
    while left < right:
        current_sum = sorted_arr[left] + sorted_arr[right]
        
        if current_sum == target:
            return True
        elif current_sum < target:
            left += 1   # Move the left pointer forward
        else:
            right -= 1  # Move the right pointer backward
            
    return False

```

### Step-by-Step Solution:

1. **Best-Case Scenario ($\Omega$):** Imagine the first and last elements immediately sum up to the exact target value (e.g., `sorted_arr = [1, 3, 5, 9]`, `target = 10`). The very first evaluation step triggers `current_sum == target`, exiting immediately.
* **Best-Case Time Complexity:** $\Omega(1)$


2. **Worst-Case Scenario ($O$):** Imagine the target pair does not exist in the array at all. The `left` and `right` pointers will start at opposite boundaries and slowly march toward each other until they meet in the middle. Collectively, they traverse the length of the list ($n$) exactly once.
* **Worst-Case Time Complexity:** $O(n)$


3. **Space Complexity:**
* **Auxiliary Space Complexity:** $O(1)$ — Only tracking two pointer indexes in memory.

## Exercise 4: Independent vs. Nested Scaling

Analyze the total time complexity of this multi-stage function.

```python
def multi_stage_processor(arr):
    n = len(arr)
    
    # Stage 1: Independent Linear Loop
    for item in arr:
        print("Registering element:", item)
        
    # Stage 2: Independent Nested Matrix Loops
    for i in range(n):
        for j in range(n):
            val = arr[i] * arr[j]

```

### Step-by-Step Solution:

1. **Break down Stage 1:** A single standalone loop runs down the length of the array. This takes $n$ execution steps $\rightarrow O(n)$.
2. **Break down Stage 2:** These loops are *not* inside Stage 1; they are written sequentially below it. However, they are nested within each other. The outer loop runs $n$ times, and the inner loop runs $n$ times per outer iteration. This takes $n \times n = n^2$ execution steps $\rightarrow O(n^2)$.
3. **Aggregate and evaluate asymptotic growth:** Total operations $T(n) = n + n^2$. Applying the rules of Asymptotic Analysis, we look at performance as $n \to \infty$. The $n^2$ term completely dominates the growth curve, making the smaller $n$ term insignificant.
4. **Final Complexities:**
* **Time Complexity:** $O(n^2)$
* **Space Complexity:** $O(1)$

## Exercise 5: The Sub-Array Slicing Trap

Analyze the time and space complexity of the following string processing script.

```python
def extract_prefixes(text_string):
    n = len(text_string)
    prefix_storage = []
    
    for i in range(1, n + 1):
        # Highlighting the slicing operation: text_string[0:i]
        chunk = text_string[0:i]
        prefix_storage.append(chunk)
        
    return prefix_storage

```

### Step-by-Step Solution:

1. **Spot the hidden loop:** A common pitfall is looking at the single explicit `for` loop and stating the runtime is $O(n)$. However, the line `chunk = text_string[0:i]` is an array slicing operation. Behind the scenes, the computer must physically copy characters one-by-one from memory to create that slice.
2. **Trace the operation steps:** * When $i = 1$, slicing copies 1 character.
* When $i = 2$, slicing copies 2 characters.
* When $i = n$, slicing copies $n$ characters.


3. **Formulate the math:** Total step count is the summation series:

$$\text{Steps} = 1 + 2 + 3 + \dots + n = \frac{n(n + 1)}{2} = \frac{1}{2}n^2 + \frac{1}{2}n$$



Dropping constants and low-order terms yields a quadratic profile.
4. **Analyze Space Complexity:** The `prefix_storage` list holds $n$ elements. However, each element is a string of variable size (averaging a length of $\frac{n}{2}$ characters). The total memory cells filled up inside RAM scales with $n \times \frac{n}{2} = \frac{1}{2}n^2$.
5. **Final Complexities:**
* **Time Complexity:** $O(n^2)$
* **Space Complexity:** $O(n^2)$ Auxiliary Space.
