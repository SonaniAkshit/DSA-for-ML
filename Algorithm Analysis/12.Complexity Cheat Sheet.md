# Complexity Cheat Sheet

Here is ultimate **Algorithmic Complexity Cheat Sheet**. Bookmark this as a quick reference for technical interviews, code reviews, and system architecture planning.

---

## 1. The Big-O Growth Hierarchy

This table ranks common time complexities from most efficient (fastest) to least efficient (slowest) as the input size ($n$) scales toward infinity.

| Complexity | Name | Efficiency | Scale Impact (If $n$ increases $100\times$) | Common Example Operation |
| --- | --- | --- | --- | --- |
| **$O(1)$** | Constant | **Excellent** | No change in execution steps. | Accessing an array element by index. |
| **$O(\log n)$** | Logarithmic | **Excellent** | Adds only a tiny, fixed handful of steps. | Binary search on a sorted list. |
| **$O(n)$** | Linear | **Good** | Takes exactly $100\times$ more steps. | A single loop searching an unsorted array. |
| **$O(n \log n)$** | Linearithmic | **Fair** | Takes roughly $200\times$ more steps. | Efficient sorting algorithms (MergeSort, QuickSort). |
| **$O(n^2)$** | Quadratic | **Poor** | Takes **$10,000\times$** more steps! | Nested loops (BubbleSort, comparing all pairs). |
| **$O(2^n)$** | Exponential | **Terrible** | Completely freezes the machine at modest $n$. | Generating all subsets of a collection. |
| **$O(n!)$** | Factorial | **Terrible** | Unusable for anything beyond tiny inputs. | Generating all permutations of a string. |

## 4. Code Pattern Recognition Rules

Use these structural layout indicators to estimate complexities instantly when reading code architecture:

### Constant Time: $O(1)$

* No loops.
* No recursive function calls.
* Straight sequential mathematical assignments or index properties.

### Logarithmic Time: $O(\log n)$

* A loop index variable is multiplied or divided by a factor (e.g., `i = i // 2` or `i *= 2`) on every iteration.
* The problem domain size gets sliced in half at each step.

### Linear Time: $O(n)$

* A loop walks sequentially from $0$ up to $n$ by adding or subtracting a fixed step (e.g., `i += 1`).
* Shifting characters via slicing patterns, parsing substrings, or making strings copies.

### Quadratic Time: $O(n^2)$

* Two loops are nested inside one another, where both loop bounds scale independently up to the value of $n$.
* Comparing every single entry in an array against every other entry in that same array.

## 5. Quick Interview Pro-Tips

* **Identify $n$ First:** Before stating a complexity, always explicitly vocalize what the parameter represents. (*"Assuming $n$ is the number of rows in our input matrix..."*)
* **Multi-Input Separations:** If a function takes two distinct arrays as parameters, do not write $O(n^2)$ for nested evaluations. Use two discrete scaling tokens: $O(A \times B)$.
* **Drop the Constants:** Always simplify configurations immediately. Never say $O(2n)$ or $O(n + 15)$. Keep only the dominant term and drop coefficients: $O(n)$.
* **Account for Hidden Steps:** Remember that built-in functions (like Python's `in` lookup for arrays, `.pop(0)`, string concatenations, or slicing actions `arr[0:k]`) are **not** free; they frequently execute $O(n)$ linear work behind the scenes.
